CHANGELOG

What we've shipped recently.

Tymr is in active development. Here's what's landed in the last few weeks — in reverse chronological order.

2026-05-17 · WEBSITE
Repositioned as the capacity system of record
The landing page now leads with the worker promise (zero-entry capture), then the manager/PMO promise (capacity, project cost, portfolio), then the executive promise (portfolio scenarios, agent workforce economics). Compliance moved to a dedicated /compliance sub-page and reframed as a byproduct of the ledger, not the headline. New 0 / 9 / 1 stats triplet. Persona page rebuilt around worker, manager & PMO, executive. UTEv1 reframed as the neutral substrate the human + AI workforce emits into.
positioning website
2026-04-19 · WEBSITE
Compliance-first homepage (superseded)
Earlier rewrite that framed Tymr around the R&D Credit + ASC 350-40 beachhead. Useful for the compliance buyer but boxed the product into a single function and audience. Replaced by the capacity-system-of-record positioning above.
positioning
2026-04-17 · OPEN SOURCE
UTEv1 v0.1 + TypeScript & Python SDKs
First public draft of the Universal Time Event spec — JSON Schema 2020-12 — plus reference client SDKs in TypeScript and Python. Includes wrapClaudeAgent (Anthropic) and TymrCallbackHandler (LangChain). Generated types, buffer/flush semantics, session/task context managers.
UTEv1 SDK
2026-04-17 · INTEGRATION
GitHub org-level webhook
Push, pull-request, and review events now flow straight into the audit ledger via a single org-level webhook. Adds the third deep-integration source system (after Jira and calendars). HMAC signature verification on the roadmap.
integration github
2026-04-16 · STRATEGY
Compliance-as-headline phase (superseded)
Earlier attempt to use the audit-ready labor-cost layer as the primary positioning. Useful for the compliance buyer but kept managers, PMOs, IT, services, and non-profits out of the door. Re-framed in May around capacity as the foundation, with compliance as a byproduct of accurate data.
positioning
2026-04-16 · PRODUCT
Org-level visibility flags & tier repack
Floor, cost-ticker, and agent features are now optional per org. Privacy-sensitive deployments can run Tymr as a pure evidence ledger without the live-floor UX. Three new tiers mapped to visibility levels.
privacy packaging
2026-04-14 · PRODUCT
Per-org integration activity config
Per-org taxonomy, canonical override rules, and preview-before-enable flow. Admins can customize which source events map to which classification, with a diff view before committing changes.
admin
