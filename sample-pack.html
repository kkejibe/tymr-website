Slide 09 · Methodology · 2-page summaryPlain English, exam-defensible

How we got from source events to qualified hours.

  • Data sources. Event-level ingest from Jira (status transitions, comments, worklogs), GitHub (commits, PRs, reviews), Google Calendar & Microsoft Calendar (meeting blocks with engineering attendees).
  • Attribution. Each event attributes time to (a) the engineer who generated it and (b) the project inferred from Jira/GitHub metadata. Overlapping events deduplicated per the "primary activity" rule.
  • § 41 classification. Four-part test applied per activity bucket: permitted purpose, technological in nature, elimination of uncertainty, process of experimentation. Rules library is auditable; tags are reviewable and overridable.
  • § 174 capitalization. Post-enactment (post-2022) software development costs flagged for 5-year amortization unless classified OpEx.
  • ASC 350-40. Application-development stage separated from preliminary project stage and post-implementation stage using commit/PR lifecycle signals.
  • Human review. Controller reviews low-confidence (<0.85) classifications weekly, ≤30 minutes. No engineer survey.
